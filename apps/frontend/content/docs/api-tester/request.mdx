## Request

In the API tester, **Request** is everything your client sends for a single API call. The UI builds one HTTP request from the tabs and sends it when you click **Send**.

### Parts of a request

| Part | Where you set it | Example |
|------|------------------|---------|
| **URL** | Built from base + path; query string from Params tab | `https://api.example.com/users?limit=10` |
| **Method** | Shown in the route header (from route meta) | `GET`, `POST`, `PUT`, `PATCH`, `DELETE` |
| **Path params** | Params tab — value substituted into path | Path `/users/:id` → `/users/42` |
| **Query params** | Params tab — key-value pairs after `?` | `?limit=10&offset=0` |
| **Body** | Body tab (when route has request meta) | JSON or form-data |
| **Content-Type** | Body tab dropdown or Headers tab | `application/json`, `multipart/form-data` |
| **Headers** | Headers tab | `Accept`, `X-Custom-Header`, etc. |
| **Authorization** | Authorization tab | Bearer token, Basic, or API key |

### Example

For a **POST** to `/users` with a JSON body:

1. **Params** — Add any query params (e.g. `dryRun=true`).
2. **Body** — Choose **application/json**, fill keys (e.g. `name`, `email`).
3. **Headers** — Add or leave default; Content-Type is set from Body.
4. **Authorization** — Set Bearer token if the API requires it.
5. Click **Send**; the **Response** panel shows status, body, and headers.

See [Request content type](/docs/api-tester/request-content-type) and [Params](/docs/api-tester/params) for details on each part.
