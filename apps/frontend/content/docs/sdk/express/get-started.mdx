## Install

Install `@tracelet/express`. It depends on `@tracelet/core` (installed automatically). The docs UI is bundled inside the package.

<CodeBlockTabs
  npm="npm install @tracelet/express"
  yarn="yarn add @tracelet/express"
  pnpm="pnpm add @tracelet/express"
  language="bash"
/>

**Peer dependency:** Express `^4` or `^5`.

## Use in code

Call `tracelet()` with your Express `app` and options. Call it **before** your routes so the middleware runs for every request.

```typescript
import express from "express";
import { tracelet } from "@tracelet/express";

const app = express();
app.use(express.json());

tracelet({
  app, // required
  environment: "local", // optional
  logFilePath: "./logs/tracelet.log", // optional
  debug: false, // optional
  traceletDocOptions: {
    uiBasePath: "/docs", // optional, default "/tracelet-docs"
    docFilePath: "./tracelet.doc.json", // optional
  },
  meta: [], // optional: TraceletMeta[] for route meta
});

// Your routes
app.use("/users", userRoutes);
app.listen(3000);
```

This enables request/response logging, optional file logging, and mounts the docs UI (API Explorer + Logs). Open `/docs` (or `/tracelet-docs` if you didn’t set `uiBasePath`) in the browser.

### Options

| Option | Default | Description |
|--------|---------|-------------|
| `app` | — | Express `Application` instance. Required. |
| `environment` | `undefined` | `"local"` or `null`. Not passing value means your code is deployed. |
| `logFilePath` | `"tracelet.log"` | Path to the log file. Logger writes here; docs UI GET `/logs` reads from it. |
| `debug` | `false` | Enable debug logging. |
| `traceletDocOptions.uiBasePath` | `"/tracelet-docs"` | Base path for the docs UI. |
| `traceletDocOptions.docFilePath` | `"tracelet.doc.json"` | Path to a JSON file with route meta for the API Explorer. |
| `meta` | `null` | `TraceletMeta[]` for route meta (in code); merged with doc file. |

### Custom UI path

To serve the docs UI at a different path, set `traceletDocOptions.uiBasePath`:

```typescript
tracelet({
  app,
  traceletDocOptions: { uiBasePath: "/api-docs" },
});
```

The UI is then available at `http://localhost:3000/api-docs`.

## Running locally

Run your app as usual. The docs UI is served by Express at the path you set (default `/tracelet-docs`). No separate dev server.

### Single server (recommended)

Start your app and open the UI (e.g. `http://localhost:3000/tracelet-docs` or `http://localhost:3000/docs`). The UI fetches route meta from the same origin; no env vars needed.

### UI on a different origin

If the docs UI is served from another port or origin, set `TRACELET_DOC_API_ROUTE` to your API base URL (e.g. `http://localhost:3000`) so the UI can call the route-meta and logs endpoints.

## Class-based usage

You can use the class and call `start()` yourself:

```typescript
import { TraceletExpress } from "@tracelet/express";

const t = new TraceletExpress({
  app, // required
  logFilePath: "./logs/app.log", // optional
  traceletDocOptions: { uiBasePath: "/docs" }, // optional
  environment: "local", // optional
  debug: false, // optional
});
t.start();
```
