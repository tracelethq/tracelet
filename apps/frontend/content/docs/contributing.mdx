## Setup

Set up the Tracelet monorepo on your machine.

### Prerequisites

- **Node.js** 18+ (20+ recommended)
- **npm**, **yarn**, or **pnpm** (use one consistently)

### Clone and install

```bash
git clone <repo-url>
cd tracelet
```

Install dependencies from the repository root:

<CodeBlockTabs
  npm="npm install"
  yarn="yarn install"
  pnpm="pnpm install"
  language="bash"
/>

This installs dependencies for all workspaces (`packages/*`, `apps/*`, `examples/*`).

## Build

Build the Tracelet packages. Order matters: core → ui → express.

### Full build (from repo root)

<CodeBlockTabs
  npm="npm run build"
  yarn="yarn build"
  pnpm="pnpm run build"
  language="bash"
/>

This runs in order:

1. **@tracelet/core** – core SDK (Logger, RouteMeta, LogReader, etc.)
2. **@tracelet/ui** – docs UI (Vite → `packages/sdk-ui/dist/`)
3. **@tracelet/express** – Express SDK (`tracelet()` + middleware + bundled docs UI); copies `sdk-ui/dist` into `dist/ui/`

### Build a single package

From root, the same command builds all. To build one workspace:

```bash
yarn workspace @tracelet/core run build
yarn workspace @tracelet/ui run build
yarn workspace @tracelet/express run build
```

Building only `@tracelet/express` requires `packages/sdk-ui/dist/` to exist first (copy step).

## Project layout

```
tracelet/
├── package.json              # Root workspace config and scripts
├── packages/
│   ├── sdk-core/             # @tracelet/core – core SDK
│   ├── sdk-express/          # @tracelet/express (tracelet() + middleware + docs UI)
│   │   ├── dist/
│   │   │   └── ui/           # Bundled docs UI (after build, for packed .tgz)
│   │   └── scripts/copy-ui.js
│   └── sdk-ui/               # @tracelet/ui – Vite + React docs UI
│       └── dist/
├── apps/
│   └── frontend/             # Docs site (Next.js)
│       └── content/docs/     # MDX docs (Introduction, SDK, API Explorer, Logs)
├── examples/
│   └── express-basic/
```

## Documentation

The docs site lives in **apps/frontend** (Next.js). Doc content is in **apps/frontend/content/docs** as MDX files.

### Run the docs site locally

From repo root:

```bash
yarn workspace frontend run dev
```

Or from `apps/frontend`: `yarn dev` (runs on port 3001).

### Edit docs

- Add or edit `.mdx` files under **apps/frontend/content/docs**. New pages appear in the sidebar automatically (see [Docs README](/docs) for structure).
- **SDK docs** – `content/docs/sdk/express/`, `content/docs/sdk/core/`
- **Logs docs** – `content/docs/logs/` (Setup, Debug option, Logs UI)
- **API Explorer** – `content/docs/api-explorer/`

Use frontmatter (`title`, `description`, `icon`, `order`) for nav and metadata. Global search (⌘K) indexes doc content and the menu.

## Pack & release

Create installable `.tgz` files for the SDKs.

<CodeBlockTabs
  npm="npm run pack:sdks"
  yarn="yarn pack:sdks"
  pnpm="pnpm run pack:sdks"
  language="bash"
/>

This will:

1. Build @tracelet/core, @tracelet/ui, @tracelet/express (in that order)
2. Pack each workspace into .tgz tarballs

Typical output: `tracelet-core-0.0.1.tgz`, `tracelet-express-0.0.1.tgz`, `tracelet-ui-0.0.1.tgz` (in each package dir or root).

## External testing

Test that the SDK works outside the monorepo (e.g. before publishing).

### Option 1: yarn link

From monorepo: build packages, then in `packages/sdk-core` and `packages/sdk-express` run `yarn link`. In your external app run `yarn link @tracelet/core` and `yarn link @tracelet/express`.

### Option 2: Install from .tgz

Run `pack:sdks`, then in your external app:

<CodeBlockTabs
  npm="npm install ./path/to/tracelet-core-0.0.1.tgz ./path/to/tracelet-express-0.0.1.tgz"
  yarn="yarn add file:./path/to/tracelet-express-0.0.1.tgz file:./path/to/tracelet-core-0.0.1.tgz"
  pnpm="pnpm add ./path/to/tracelet-core-0.0.1.tgz ./path/to/tracelet-express-0.0.1.tgz"
  language="bash"
/>

Verify imports, types, and that the docs UI loads at your `uiBasePath`. Use yarn link for daily dev; use .tgz before publishing.
